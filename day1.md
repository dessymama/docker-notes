1. What is a Docker? Docker packages an application and everything it needs, its codes, libraries,and dependencies into a container that can run on a laptop,test server,and a production server.
2. What is a Container? A container contains an application and all the files, libraries, and settings a Docker needs to run. 
3. Two advantages of Docker
Consistency: Applications run the same way in development, testing, and production, reducing "it works on my machine" problems.
Speed and efficiency: Containers start quickly and use fewer system resources than virtual machines, allowing you to run many containers on the same server.
4. Why DevOps engineers use Docker
DevOps engineers use Docker because it:
Makes applications easy to build, test, and deploy consistently across different environments.
Works well with CI/CD pipelines, enabling faster and more reliable software releases.
Simplifies application deployment and scaling.
Makes it easier for developers and operations teams to collaborate using the same containerized application.
